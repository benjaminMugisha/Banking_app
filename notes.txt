👉 admin:
so admins don't need to have an account thats all we're going to change.

we currently have an architecture where admin is just another user with admin role. 
we are enforcing admin to have an account which is not neccessary since they dont do transactions. 
users is equal to authentication+role and Account should be equal to banking data. 1-1 model.

we have a mapping thats ok if account=null because:
    1. no database constraint forcing it. 
    2. no @NotNull in validation.
    3. no optional=false on both Users and Account entity class.
so JPA treats this as Users MAY have an account. 
P.S the Account entity class owns the relationship because of the line: @JoinColumn(name="user_id")
that means users table has no foreign key and account table references users. 
so no account row => no contstraint violated. so admins without accounts are 100% safe.
